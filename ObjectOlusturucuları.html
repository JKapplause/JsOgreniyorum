Yukarıda görebileceğiniz gibi, JavaScript ilkel veri türlerinin nesne sürümleri String, Numberve Boolean. Ancak karmaşık nesneler oluşturmak için hiçbir neden yoktur.

AYRICA:

{}Bunun yerine nesne değişmez değerleri kullanın new Object().
""Bunun yerine dize değişmez değerleri kullanın new String().
12345Bunun yerine sayı değişmez değerleri kullanın new Number().
true / falseBunun yerine boole değişmez değerleri kullanın new Boolean().
[]Bunun yerine dizi değişmezlerini kullanın new Array().
/()/Bunun yerine desen değişmez değerleri kullanın new RegExp().
() {}Bunun yerine işlev ifadelerini kullanın new Function().

Normalde, dizeler ilkel olarak oluşturulur: var firstName = "John"
Ancak dizeler, newanahtar kelime kullanılarak nesne olarak da oluşturulabilir :var firstName = new String("John")
Dizelerin neden JS Dizeleri bölümünde nesne olarak oluşturulmaması gerektiğini öğrenin .

Normalde sayılar ilkel olarak oluşturulur: var x = 123
Ancak sayılar, newanahtar kelime kullanılarak nesne olarak da oluşturulabilir :var x = new Number(123)
JS Numaraları bölümünde sayıların neden nesne olarak oluşturulmaması gerektiğini öğrenin .

Normalde, booleans ilkel olarak oluşturulur: var x = false
Ancak booleans newanahtar kelimesini kullanarak nesne olarak da oluşturulabilir :var x = new Boolean(false)
Booleanların neden JS Booleans bölümünde nesne olarak oluşturulmaması gerektiğini öğrenin .
